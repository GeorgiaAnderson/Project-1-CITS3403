Purpose of HTML

HTML, which stands for Hypertext Markup Language, is a scripting language used to define the structure and layout of webpages displayed on web browsers. These web pages are a series of pages that are connected by hyperlinks. HTML consists of a series of elements which tell the browser how to display the content. These elements are represented by tags such as ‘heading’ and ‘paragraph’, however the browser does not display these tags rather they are used to render the content of the web page.

History of HTML

1993: HTML1.0

Created by physicist Tim Berners-Lee, the first version HTML was a system designed to allow scientists to share papers with one another using electronic networking methods. He created a simple version of the language SGML which allowed people who weren’t specialised in SGML to share their technical documents. HTML was particualy useful as it made it easy to share documents through hyperlinks. The language was limited in terms of styling, only contained basic elements like text controls and images, and it was only supported by 2 browsers, lynx and mosaic.

1995: HTML 2.0

The second version of HTML was realised in 1995 by IETF (Internet Engineering Task Force). HTML 2 provided the first set of official standards for HTML and aimed to support more browsers, improve presentation, improve styling and include more useful elements. It included all the features of HTML 1, but also introduced new styling features to change the webpage background, text colour and text interface. It also introduced queries like key word searches and basic forms for data transfer from computer to server.

1995: HTML 3.0

As a draft standard released by W3C, HTML 3.0 contained many new features including tables, layout for math formulas, text-wrap and new browser features such as banner and fig however it was not addopted in full by any browser. HTML 3.0 was released during a time where browsers were at ‘war’ to implement the best features and introduced proprietary elements that only worked on their browsers causing the language to become very fragmented. It was important to stop the fragmentation by releasing a HTML standard that worked on all browsers. This caused W3C to abandon HTML 3.0 and begin to work on HTML 3.2.

1997: HTML 3.2

Released by W3C as the official industry wide standard, HTML 3.2 added support for style sheets and included improved form features and the script tag so JavaScript could be applied easily.

1999: HTML 4.01

As further development, HTML 4.01 focussed on separating presentation and styling information from the actual content by marking many HTML tags as outdated, and making the use of style sheets necessary. It also cleaned-up and improved many elements from previous versions including forms, images, tables and objects.

2008: HTML 5.0

Developed by WHATWG and W3C, HTML 5.0 made HTML simpler to write, provided more features and had better error handling. Some of the new features include canvas, video, audio and the support of multimedia elements.

Key HTML element descriptions

<a> Anchor tag </a>

Defines a hyperlink which is used to link web pages, email addresses, locations in the same page. Its most important attributes are the “href” attribute which states the URL that the hyperlink points too and the “target” attribute which defines how a link will be opened.

<!-- This anchor tag defines a text link with a target of "_blank" which will open the CSS page of this web page in a new browser window-->

<a href=csspage.html> CSS Page </a>

<div> Div tag </div>

It’s a block level element that groups HTML elements together enabling CSS and JS to be applied too many elements at once.

<h5>This is a subtitle within a div </h5>

<p>This is a paragraph within a div </p>

</div>
